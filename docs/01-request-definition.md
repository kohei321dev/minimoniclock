# 要求定義

## 背景

- [事実] ユーザーは JAPANNEXT の 7.8 インチ級サブディスプレイ `JN-MD-IPS7842` を購入した。
- [事実] JAPANNEXT 公式ページでは、`JN-MD-IPS7842` は 7.8 インチ IPS パネル、解像度 `400x1280`、縦横使用可能、Windows のみに対応、スピーカー非搭載と説明されている。
- [事実] 既存の時計アプリでは、この小型サブディスプレイにちょうど合う表示、縦横両対応、ポモドーロ、アラームを同じ画面で満たすものが見つかっていない。
- [事実] 稼働環境はユーザーの Windows 端末である。
- [未検証] 対象ディスプレイの Windows 上の論理サイズ、DPI、表示スケーリング設定、回転時の座標系は未確認。

## 作りたいもの

Windows デスクトップ上で動作し、小型サブディスプレイに常時表示できる時計アプリを作る。

アプリは、次の価値を提供する。

- 小型ディスプレイの縦向き・横向きの両方で、時刻がひと目で分かる。
- 時計は 24 時間表記のデジタル表示にする。
- 同じ画面から、最小限の操作でポモドーロタイマーを開始、停止、リセットできる。
- ログ記録は行わず、現在のカウントダウン状態を明確に表示する。
- アラーム時刻を設定し、到達時に画面で分かる通知を出す。
- タイマーやアラーム完了時に、画面上で `0` または完了状態を大きく表示できる。

ここでの `JUSTFIT` は、固定ピクセル画像を合わせることではなく、`400x1280` または `1280x400` 相当の表示領域で、時刻や残り時間が切れず、余白が過剰にならず、視認性を優先して画面に収まる状態と定義する。

## 対象ユーザー

- 主ユーザー: Windows 端末で作業し、サブディスプレイを時計・作業タイマー専用表示として使いたい本人。
- 想定利用場所: デスク上、作業中に視線を少し動かすだけで見える小型サブディスプレイ。
- 想定操作: メインディスプレイまたはサブディスプレイ上で、マウスまたはキーボードを使う。

## 利用シナリオ

### 時計常時表示

1. Windows 起動後、アプリを起動する。
2. アプリウィンドウを `JN-MD-IPS7842` に配置する。
3. 必要に応じて縦向きまたは横向き表示に切り替える。
4. 画面全体に近い領域で、24 時間表記の現在時刻を表示する。

### ポモドーロ

1. 時計画面上でポモドーロモードを選ぶ。
2. ワンクリックまたは少ない操作で 25 分などの作業タイマーを開始する。
3. カウントダウン中は残り時間が大きく表示される。
4. 完了時は `0` または完了状態が明確に表示される。
5. ログは保存しない。

### アラーム

1. 時計画面上でアラーム時刻を設定する。
2. 設定後は時計表示を維持する。
3. 指定時刻に達したら、画面上で分かる通知を出す。
4. 必要に応じて停止、スヌーズ、リセットを行う。

## 成功条件

### Phase 1

- サブディスプレイ上で、横向きでも縦向きでも時刻が読みやすい。
- 24 時間表記のデジタル時計が常時表示できる。
- Windows 端末上でローカルに動作し、外部サービスやアカウントを必要としない。
- ログ保存や外部通信を行わない。

### Phase 2 以降

- 時計、ポモドーロ、アラームの主要操作が同じ画面から完結する。
- ログ保存なしで、現在のタイマー状態だけが扱われる。
- 完了時の `0` 表示または完了表示が、離席気味でも判別できる。

## スコープ外

- 作業ログ、履歴、統計、日報出力。
- クラウド同期。
- モバイルアプリ。
- 複数ユーザー管理。
- カレンダー連携。
- Web サービスとしての公開。

## ステークホルダー観点メモ

- Requirements PM: 目的は「小型サブディスプレイで常時見やすい時計・タイマーを使うこと」。最初から多機能化しすぎない。
- UX Designer: 小型画面では機能説明よりも、視認性、操作数、誤操作防止が重要。
- Product Engineer: MVP は表示と基本操作に絞る。ログ、設定永続化、配布形式は段階的に扱う。技術候補は WPF / .NET を第一候補にする。
- Architecture Reviewer: ディスプレイ依存値をコードに固定せず、ウィンドウサイズと向きに応じてレイアウトする。
- Ops / Security Reviewer: ローカルアプリで外部送信なしを基本にし、公開 GitHub には個人環境情報や secret を含めない。配布方式、rollback、報告テンプレートは実装前に整理する。
