# 要件定義

## 前提

- [事実] 対象 OS は Windows。
- [事実] JAPANNEXT 公式ページでは、`JN-MD-IPS7842` は `400x1280` 解像度、縦横使用可能、Windows のみに対応、スピーカー非搭載と説明されている。
- [事実] ログ記録は MVP では不要。
- [事実] 時計は 24 時間表記のデジタル表示にする。
- [未検証] 対象ディスプレイ `JN-MD-IPS7842` の Windows 上の論理サイズ、DPI、Windows 表示スケール、回転時の座標系は未確認。

## 機能要件

### FR-001 時計表示

- 現在時刻を 24 時間表記で表示する。
- 初期表示は `HH:mm` とする。
- 秒表示 `HH:mm:ss` は設定または表示モードとして検討可能にする。
- 日付や曜日は MVP では任意表示とする。

### FR-002 縦横対応

- ウィンドウのアスペクト比に応じて、縦向き・横向きのレイアウトを切り替える。
- 画面回転またはウィンドウサイズ変更時に、文字が画面外へはみ出さない。
- 縦向きでは時刻を縦長画面に収め、横向きでは横長画面の幅を活かして表示する。

### FR-003 サブディスプレイ常時表示

- ウィンドウを対象ディスプレイ上に配置して使える。
- 全画面またはボーダーレス表示を提供する。
- 常に手前表示は MVP で採用候補とするが、他作業の邪魔になる場合に解除できるようにする。

### FR-004 ポモドーロタイマー

- 同じ画面からポモドーロタイマーを開始、停止、一時停止、リセットできる。
- 既定値は作業 25 分、休憩 5 分を候補とする。
- MVP では作業タイマー単体から始め、休憩切替や複数セットは後続でよい。
- カウントダウン中は残り時間を大きく表示する。
- 完了時は `0` または完了状態を大きく表示する。
- 完了ログは保存しない。

### FR-005 カウントダウン表示

- 任意またはプリセット時間のカウントダウンを表示できる。
- 残り時間が 0 に到達したら、画面上の表示を明確に変える。
- 音や通知は設定可能な拡張候補とし、MVP では画面表示を優先する。

### FR-006 アラーム

- 指定時刻のアラームを設定できる。
- アラーム到達時に画面上で分かる表示を出す。
- 停止操作を提供する。
- 音を出す場合は PC 側の音声出力を使う。対象ディスプレイ自体にスピーカーはない。
- スヌーズは MVP では任意とし、未対応リストに残す。

### FR-007 操作 UI

- 時計、ポモドーロ、アラームを同一画面から切り替えられる。
- 小型画面で誤操作しにくい大きさのボタンを使う。
- 操作数は、ポモドーロ開始まで 1 から 2 操作を目標にする。
- キーボードショートカットは後続候補とする。

### FR-008 設定

- MVP では最低限、表示モード、常に手前表示、ポモドーロ時間、アラーム時刻を扱う。
- 設定の永続化は初期実装では任意。永続化する場合も、ログや履歴ではなく現在の設定値だけに限定する。

### FR-009 保存データ境界

- 保存しないもの: ポモドーロ実績、操作履歴、アラーム発火履歴、利用時間履歴、画面情報、ユーザー名、端末識別子、クラッシュログ、自動送信データ。
- 保存してよい候補: UI 設定、表示位置、表示先ディスプレイ、テーマ、ポモドーロ設定、アラーム設定。
- 設定を保存する場合でも、履歴やログとは分離して扱う。

## 非機能要件

### NFR-001 視認性

- 小型ディスプレイ上で、1 メートル程度離れても時刻が読めることを目標にする。
- 背景と文字のコントラストを高くする。
- フォントサイズは固定値ではなく、ウィンドウサイズに応じて自動調整する。

### NFR-002 レスポンシブレイアウト

- 幅、高さ、アスペクト比に応じて UI が破綻しない。
- 縦向き・横向きの最小想定サイズは実機確認後に確定する。

### NFR-003 安定性

- 長時間表示しても時計更新が止まらない。
- PC のスリープ復帰後に時刻表示が復帰する。
- タイマー中にスリープした場合の扱いを定義する。

### NFR-004 軽量性

- サブディスプレイ常時表示を前提に、CPU 使用率とメモリ使用量を抑える。
- 秒表示やアニメーションは負荷と視認性のバランスを見て採用する。

### NFR-005 ローカル完結

- MVP ではネットワーク通信を行わない。
- アカウント、クラウド、外部 API を不要にする。
- GitHub 公開時に secret や個人環境値を含めない。
- 自動クラッシュレポート送信は MVP では行わない。

### NFR-006 配布・起動

- Windows 上で起動できる形式にする。
- 初期対応 OS は Windows 11 を第一候補にする。Windows 10 対応は技術選定後に確認する。
- 将来的にはインストーラー、ポータブル exe、自動起動のいずれかを検討する。
- MVP では開発者環境から起動できる状態でもよいが、配布方式は未対応リストで管理する。

## 受け入れ条件

- AC-001: 横長ウィンドウで `HH:mm` が画面内に大きく表示される。
- AC-002: 縦長ウィンドウで `HH:mm` が画面内に大きく表示される。
- AC-003: ウィンドウサイズ変更時に文字やボタンが画面外へはみ出さない。
- AC-004: ポモドーロタイマーを 2 操作以内で開始できる。
- AC-005: ポモドーロの残り時間がカウントダウン表示される。
- AC-006: タイマー完了時に `0` または完了状態が大きく表示される。
- AC-007: アラーム時刻を設定し、到達時に画面上で通知される。
- AC-008: ログ、履歴、統計データが保存されない。
- AC-009: ネットワーク接続なしで主要機能が動作する。
- AC-010: `400x1280` 相当と `1280x400` 相当の表示領域で、主要表示が欠けない。
- AC-011: アラーム音を使う場合、ディスプレイ内蔵スピーカーではなく PC 側音声出力を前提にする。

## Phase 1 MVP スコープ

- 24 時間デジタル時計。
- 縦横レスポンシブ表示。
- ボーダーレスまたは全画面表示。
- ログ保存なし。
- 外部通信なし。

## Phase 2 拡張スコープ

- ポモドーロ用カウントダウン開始、一時停止、リセット。
- タイマー完了時の `0` または完了表示。
- 単発アラーム設定と画面通知。
- 設定保存。

## 運用要件

- 起動しない場合、画面外に出た場合、音が鳴らない場合、タイマーがずれる場合の確認手順を README または Runbook に残す。
- 配布する場合は、直前バージョンへ戻す rollback 手順を用意する。
- Issue template を作る場合、ログ添付を必須にせず、OS version、アプリ version、再現手順、任意スクリーンショットだけを求める。
- GitHub Releases を使う場合、公開する成果物と含めないファイルを明確にする。

## 後続候補

- 秒表示切替。
- 日付、曜日表示。
- スヌーズ。
- 複数アラーム。
- 休憩タイマー自動切替。
- Windows 起動時の自動起動。
- 対象ディスプレイの自動選択。
- 設定保存。
- 音声通知。
- テーマ切替。
